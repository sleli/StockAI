## User Story

**Epic:** EP-004 — Alert di Prezzo
**Priorità:** MEDIUM | **Story Points:** 3

**Story**
As Marco (investitore retail),
I want vedere chiaramente in dashboard quali alert si sono attivati e lo stato di ciascuno,
so that possa reagire rapidamente ai movimenti di prezzo significativi.

**Criteri di Accettazione**
- [ ] I titoli con alert attivati sono evidenziati visivamente nella watchlist (badge, bordo colorato o icona)
- [ ] Una sezione alert nella dashboard elenca tutti gli alert dell'utente con stato: attivo, attivato (triggered), disattivato
- [ ] L'alert attivato mostra il prezzo che ha causato il trigger e il timestamp
- [ ] L'utente può confermare (dismiss) un alert attivato per rimuovere l'evidenziazione

## Soluzione Tecnica

> **Architetto:** Leonardo 📐

### Approccio Scelto

Estendere WatchlistPanel con evidenziazione visiva basata sui triggered alerts, integrare AlertList nella dashboard con stati e dettagli, aggiungere API dismiss per conferma alert.

### Motivazione

Si costruisce sui componenti e API già implementati in US-012 (AlertList, CRUD API, polling evaluate). L'evidenziazione nel WatchlistPanel sfrutta i dati `triggeredAlerts` già esposti dal polling hook. Il dismiss è un'estensione naturale del PATCH status esistente.

### Componenti Impattati

| Componente | Tipo Modifica | Descrizione |
|---|---|---|
| `src/components/watchlist-panel.tsx` | Modifica | Aggiungere badge/bordo per titoli con alert triggered |
| `src/app/dashboard/page.tsx` | Modifica | Integrare sezione alert con AlertList esteso |
| `src/components/alert-list.tsx` | Modifica | Mostrare triggeredPrice, triggeredAt, pulsante dismiss |
| `src/app/api/alerts/[id]/route.ts` | Modifica | Supportare status "dismissed" nel PATCH |
| `src/components/alert-badge.tsx` | Nuovo | Badge componente per evidenziazione alert triggered |

### Modifiche al Data Model

Nessuna modifica allo schema — il campo `status` del modello Alert supporta già stringhe. Si aggiunge il valore "dismissed" come convenzione applicativa.

### Modifiche alle API

**`PATCH /api/alerts/[id]`** — Estensione
- Body: `{ status: "dismissed" }` (nuovo valore ammesso)
- L'alert dismissed non viene più evidenziato nella watchlist
- Risposta 200: `{ alert: Alert }`

### Modifiche al Frontend

**`WatchlistPanel`:** Per ogni titolo nella watchlist, verificare se esiste un alert triggered. Se sì, mostrare un badge/icona e applicare un bordo colorato (es. arancione). Utilizzare i dati `triggeredAlerts` dal polling hook.

**`AlertBadge`:** Nuovo micro-componente che mostra un'icona campanella con stato colorato (arancio = triggered, verde = active, grigio = disabled/dismissed).

**`AlertList` (esteso):** Aggiungere colonne/dettagli per gli alert triggered:
- Prezzo che ha causato il trigger (`triggeredPrice`)
- Timestamp del trigger (`triggeredAt`) formattato
- Pulsante "Conferma/Dismiss" per alert triggered

**`Dashboard page`:** Aggiungere una sezione dedicata "Alert" sotto la watchlist, con l'AlertList filtrato per stati rilevanti.
