## User Story

**Epic:** EP-002 — Quotazioni Near-Real-Time
**Priorità:** HIGH | **Story Points:** 3

**Story**
As Marco (investitore retail),
I want vedere prezzo corrente, variazione assoluta, variazione percentuale e timestamp per ogni titolo in watchlist,
so that possa capire al volo come si stanno muovendo i miei titoli.

**Criteri di Accettazione**
- [ ] Ogni titolo in watchlist mostra prezzo corrente, variazione assoluta (+/-), variazione percentuale e timestamp ultimo aggiornamento
- [ ] Variazioni positive sono evidenziate in verde, negative in rosso
- [ ] Il timestamp mostra chiaramente quando il dato è stato aggiornato (es. "14:32:05" o "2 min fa")
- [ ] Al caricamento della dashboard, le quote vengono recuperate dal server per tutti i titoli in watchlist

## Soluzione Tecnica

> **Architetto:** Leonardo 📐

### Approccio Scelto

Aggiungere metodo `getQuotes(symbols)` al provider per batch fetching + endpoint `/api/quotes/batch` + estendere WatchlistPanel per mostrare i dati di prezzo con colorazione.

### Motivazione

Il batch fetching riduce il numero di chiamate API (critico con il rate limit del free tier). Il Twelve Data API supporta multiple symbols separati da virgola in una singola chiamata `/quote?symbol=AAPL,MSFT,TSLA`.

### Componenti Impattati

| Componente | Tipo Modifica | Descrizione |
|---|---|---|
| `src/lib/market-data/provider.ts` | Modifica | Aggiungere metodo `getQuotes(symbols: string[])` |
| `src/lib/market-data/twelve-data.ts` | Modifica | Implementare batch quote con multi-symbol |
| `src/app/api/quotes/batch/route.ts` | Nuovo | GET: quote batch per lista di ticker |
| `src/components/watchlist-panel.tsx` | Modifica | Mostrare prezzo, variazione, colore, timestamp |

### Modifiche al Data Model

Nessuna modifica allo schema.

### Modifiche alle API

**`GET /api/quotes/batch?symbols=AAPL,MSFT,TSLA`**
- Auth: richiesta (le quote sono per la watchlist dell'utente)
- Parametro: `symbols` (string, ticker separati da virgola)
- Risposta 200: `{ quotes: { [symbol: string]: NormalizedQuote } }`
- Risposta 400: symbols mancanti
- Risposta 502/429: errore provider

### Modifiche al Frontend

- WatchlistPanel fetch quote al mount dopo aver caricato gli items
- Ogni item mostra: ticker, nome, prezzo (font mono), variazione assoluta, variazione %, timestamp
- Colore: variazione positiva → verde (#12392c), negativa → rosso (#a13d30)
- Formato: prezzo con 2 decimali, variazione con segno (+/-), percentuale con 2 decimali
- Timestamp: formato relativo ("2 min fa") o orario ("14:32:05")
