## Soluzione Tecnica

> **Architetto:** Leonardo

### Approccio Scelto

Estendere il `MarketDataProvider` con un metodo `getNews(symbol)` che restituisce un array di notizie normalizzate. Implementare il metodo nel provider Twelve Data. Creare un API endpoint `/api/stocks/[symbol]/news` e un componente `StockNews` da integrare nella pagina dettaglio.

### Motivazione

Estendere il provider esistente mantiene la coerenza architetturale e consente di cambiare fonte dati in futuro senza modificare il frontend. L'endpoint dedicato separa il recupero notizie dalla quote, evitando di appesantire le chiamate esistenti.

### Componenti Impattati

| Componente | Tipo Modifica | Descrizione |
|---|---|---|
| `src/lib/market-data/types.ts` | Modifica | Aggiungere interfaccia `NormalizedNewsItem` |
| `src/lib/market-data/provider.ts` | Modifica | Aggiungere `getNews()` all'interfaccia `MarketDataProvider` |
| `src/lib/market-data/twelve-data.ts` | Modifica | Implementare `getNews()` con chiamata all'endpoint news |
| `src/app/api/stocks/[symbol]/news/route.ts` | Nuovo | GET handler per notizie di un simbolo |
| `src/components/stock-news.tsx` | Nuovo | Componente lista notizie |
| `src/app/stocks/[symbol]/page.tsx` | Modifica | Integrare `StockNews` nella pagina dettaglio |

### Modifiche al Data Model

Nessuna. Le notizie sono dati transitori recuperati in tempo reale dal provider, non persistiti nel database.

### Modifiche alle API

**`GET /api/stocks/[symbol]/news`** — Recupera notizie per un simbolo
- Nessuna auth richiesta
- Parametri URL: `symbol` (obbligatorio, dal path)
- Logica: chiama `provider.getNews(symbol)`, limita a 5 risultati
- Risposta 200:
  ```json
  {
    "symbol": "AAPL",
    "news": [
      {
        "title": "Apple Reports Record Q1 Earnings",
        "source": "Reuters",
        "publishedAt": "2026-03-19T14:30:00Z",
        "url": "https://reuters.com/...",
        "snippet": "Apple Inc. reported..."
      }
    ]
  }
  ```
- Risposta 200 (nessuna notizia): `{ "symbol": "AAPL", "news": [] }`
- Risposta 400: simbolo mancante o non valido
- Risposta 502/429: errore provider

### Modifiche al Frontend

**Componente `StockNews`:**
- Client component che riceve `symbol` come prop
- Fetch `/api/stocks/[symbol]/news` al mount
- Mostra fino a 5 card notizie, ciascuna con:
  - Titolo (link cliccabile all'articolo originale, `target="_blank"`)
  - Fonte e data pubblicazione (formattata relativamente, es. "2 ore fa")
  - Snippet/descrizione breve (se disponibile)
- Stato di caricamento con skeleton
- Stato vuoto: messaggio "Nessuna notizia disponibile per questo titolo"
- Stato errore: messaggio generico con possibilita di riprovare
- Layout con Card shadcn/ui, coerente con il design system

**Integrazione pagina dettaglio:**
- Aggiungere `StockNews` sotto il grafico/dati principali nella pagina `/stocks/[symbol]`
- Sezione con titolo "Notizie recenti"

**Descrizione:**
GET handler che:
1. Estrae `symbol` dai parametri del path
