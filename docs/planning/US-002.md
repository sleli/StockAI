## User Story

**Epic:** EP-001 — Ricerca Simboli e Watchlist
**Priorità:** HIGH | **Story Points:** 3

**Story**
As Marco (investitore retail),
I want cercare un titolo per ticker o nome azienda,
so that possa trovare rapidamente i simboli da aggiungere alla mia watchlist.

**Criteri di Accettazione**
- [ ] L'endpoint /api/search?q=<query> restituisce risultati normalizzati dal provider con ticker, nome e mercato
- [ ] Il componente di ricerca mostra i risultati in tempo reale con debounce (300ms minimo)
- [ ] I risultati mostrano chiaramente ticker, nome completo e exchange di appartenenza
- [ ] Una ricerca senza risultati mostra un messaggio chiaro ("Nessun titolo trovato")
- [ ] La ricerca funziona sia per ticker esatto (AAPL) che per nome parziale (App...)

## Analisi dei Requisiti

> **Analista:** Emanuele 🔎

### Scope

**Incluso:**
- Endpoint backend `/api/search?q=<query>` che interroga il provider Twelve Data (endpoint `/symbol_search`)
- Componente frontend di ricerca con input, debounce (≥300ms) e visualizzazione risultati
- I risultati mostrano: ticker, nome completo, exchange
- Gestione caso vuoto ("Nessun titolo trovato")
- Supporto ricerca per ticker esatto e nome parziale

**Escluso:**
- Aggiunta del titolo alla watchlist (US-003)
- Caching dei risultati di ricerca nel database
- Suggerimenti/ticker popolari (US-003 empty state)

### Requisiti Impliciti

- L'interfaccia `MarketDataProvider` deve essere estesa con un metodo `searchSymbol()` — oggi ha solo `getQuote()`
- Serve un tipo `SymbolSearchResult` per normalizzare la risposta del provider
- Il debounce deve evitare chiamate API eccessive (rate limit free tier Twelve Data: 8 req/min)
- Il componente di ricerca è un Client Component (necessita interattività)
- La ricerca non richiede autenticazione (coerente con `/api/quotes`)
- In caso di errore del provider, viene mostrato un messaggio di errore generico

### Assunzioni

- L'endpoint Twelve Data `/symbol_search` è disponibile nel piano gratuito
- I risultati di ricerca restituiscono almeno: symbol, instrument_name, exchange
- Il componente espone un callback `onSelect` per integrazione futura (US-003)

### Modifiche al Data Model

Nessuna modifica richiesta. Il modello `Symbol` esiste già in schema.prisma e verrà popolato in US-003 quando l'utente aggiunge un titolo alla watchlist.

### Modifiche alle API

**`GET /api/search?q=<query>`**
- Parametri: `q` (string, obbligatorio, minimo 1 carattere)
- Risposta 200: `{ results: SymbolSearchResult[] }`
- Risposta 400: `{ error: "Query parameter 'q' is required" }`
- Risposta 502: `{ error: "Market data provider error" }`
- Risposta 429: `{ error: "Rate limit exceeded" }`

**Formato `SymbolSearchResult`:**
```ts
{
  symbol: string;      // "AAPL"
  name: string;        // "Apple Inc"
  exchange: string;    // "NASDAQ"
  type: string;        // "Common Stock"
  country: string;     // "United States"
}
```

### Modifiche al Frontend

**Componente `SymbolSearch`** (Client Component):
- Input controllato con debounce 300ms (setTimeout/clearTimeout)
- Stati: `query`, `results`, `isLoading`, `error`
- Fetch verso `/api/search?q=...` al termine del debounce
- Rendering condizionale: loading spinner, lista risultati, messaggio vuoto, messaggio errore
- Prop `onSelect?: (result: SymbolSearchResult) => void` per integrazione futura
- Stile coerente con il design system del mockup (font serif/sans, colori paper/ink/green)
