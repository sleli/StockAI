## User Story

**Epic:** EP-001 — Ricerca Simboli e Watchlist
**Priorità:** HIGH | **Story Points:** 5

**Story**
As Marco (investitore retail),
I want creare la mia watchlist personale e aggiungervi titoli dalla ricerca,
so that possa avere tutti i titoli che mi interessano in un'unica schermata.

**Criteri di Accettazione**
- [ ] La dashboard mostra la watchlist personale dell'utente autenticato (estende la dashboard protetta esistente)
- [ ] Se la watchlist è vuota, viene mostrato un empty state con ticker suggeriti (es. AAPL, MSFT, TSLA) e call-to-action per la ricerca
- [ ] L'utente può aggiungere un titolo dalla ricerca con un click/tap e il titolo appare nella watchlist senza reload
- [ ] I titoli duplicati non possono essere aggiunti (feedback chiaro se si tenta)
- [ ] La watchlist è persistente tra sessioni (salvata su database)

## Analisi dei Requisiti

> **Analista:** Emanuele 🔎

### Scope

**Incluso:**
- Ristrutturazione dashboard esistente per mostrare la watchlist personale
- Creazione automatica di una watchlist di default ("La mia Watchlist") al primo accesso
- Integrazione del componente `SymbolSearch` (US-002) per cercare e aggiungere titoli
- Empty state con ticker suggeriti (AAPL, MSFT, TSLA) e call-to-action
- Aggiunta titolo con optimistic update (appare subito senza reload)
- Prevenzione duplicati con feedback visivo ("Titolo già in watchlist")
- Persistenza su database via modelli Watchlist, WatchlistItem, Symbol

**Escluso:**
- Visualizzazione quote/prezzi nella watchlist (US-006)
- Rimozione titoli dalla watchlist (US-004)
- Ordinamento watchlist (US-005)

### Requisiti Impliciti

- API endpoints protetti con autenticazione Supabase (401 se non autenticato)
- Upsert del `Symbol` nel database al momento dell'aggiunta (il simbolo potrebbe non esistere)
- La constraint `@@unique([watchlistId, symbolId])` in Prisma previene duplicati lato DB
- Il Prisma error P2002 (unique constraint violation) viene mappato a HTTP 409
- Lookup utente nel DB tramite `supabaseId` dall'utente Supabase autenticato

### Assunzioni

- Il componente `SymbolSearch` di US-002 è già implementato e disponibile
- La watchlist di default ha nome "La mia Watchlist" e viene creata automaticamente
- I ticker suggeriti nell'empty state sono hardcoded (AAPL, MSFT, TSLA)

## Soluzione Tecnica

> **Architetto:** Leonardo 📐

### Approccio Scelto

Due API endpoints protetti (GET watchlist, POST add item) + componente client `WatchlistPanel` che orchestra la visualizzazione e l'interazione. La dashboard rimane un server component che delega l'interattività al client component.

### Motivazione

Separare la logica di presentazione (server) dalla logica interattiva (client) segue il pattern Next.js App Router. Le API REST mantengono coerenza con gli endpoint esistenti (`/api/quotes`, `/api/search`).

### Componenti Impattati

| Componente | Tipo Modifica | Descrizione |
|---|---|---|
| `src/app/api/watchlist/route.ts` | Nuovo | GET: restituisce watchlist con items e symbols |
| `src/app/api/watchlist/items/route.ts` | Nuovo | POST: aggiunge titolo alla watchlist |
| `src/components/watchlist-panel.tsx` | Nuovo | Client component: lista titoli, empty state, integrazione search |
| `src/app/dashboard/page.tsx` | Modifica | Ristrutturazione per integrare WatchlistPanel |

### Modifiche al Data Model

Nessuna modifica allo schema — i modelli Watchlist, WatchlistItem e Symbol esistono già.

### Modifiche alle API

**`GET /api/watchlist`**
- Auth: richiesta (Supabase session via cookies)
- Auto-crea watchlist di default se l'utente non ne ha una
- Lookup utente: `supabaseId` → User → Watchlist con items e symbols
- Risposta 200: `{ watchlist: { id, name, items: [{ id, sortOrder, symbol: { ticker, name, exchange } }] } }`
- Risposta 401: `{ error: "Unauthorized" }`

**`POST /api/watchlist/items`**
- Auth: richiesta
- Body: `{ ticker: string, name: string, exchange: string }`
- Logica: upsert Symbol (by ticker) → crea WatchlistItem
- Risposta 201: `{ item: { id, symbol: { ticker, name, exchange } } }`
- Risposta 409: `{ error: "Titolo già in watchlist" }`
- Risposta 400: `{ error: "Missing required fields" }`
- Risposta 401: `{ error: "Unauthorized" }`

### Modifiche al Frontend

**`WatchlistPanel` (Client Component):**
- Fetch GET `/api/watchlist` al mount con `useEffect`
- Stato: `items`, `isLoading`, `error`, `duplicateError`
- Empty state: card con ticker suggeriti come chip cliccabili + CTA "Cerca un titolo"
- Lista titoli: card per ogni titolo con ticker (bold, mono) e nome
- Integra `SymbolSearch` con `onSelect` che chiama POST `/api/watchlist/items`
- Optimistic update: aggiunge il titolo allo stato locale prima della risposta server
- Rollback su errore, mostra toast/feedback per duplicati

**Dashboard (Server Component):**
- Mantiene header con avatar, nome, email e sign out
- Aggiunge `<WatchlistPanel />` sotto il profilo utente
- Layout più ampio per accomodare la watchlist
