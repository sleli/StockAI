## User Story

**Epic:** EP-003 — Dettaglio Titolo
**Priorità:** HIGH | **Story Points:** 5

**Story**
As Giulia (studentessa di finanza),
I want aprire una pagina di dettaglio per un titolo e vedere lo storico breve dei prezzi,
so that possa osservare l'andamento nel tempo e collegarlo a eventi di mercato.

**Criteri di Accettazione**
- [ ] La pagina /stocks/[symbol] mostra nome, ticker, prezzo corrente, high/low del giorno e variazione
- [ ] Un grafico compatto mostra lo storico intraday o degli ultimi 5 giorni lavorativi
- [ ] L'utente può selezionare il periodo visualizzato (intraday, 1 settimana, 1 mese)
- [ ] I dati storici sono recuperati dal server tramite l'adapter market-data
- [ ] La pagina è raggiungibile sia dalla watchlist che da URL diretto

## Analisi dei Requisiti

> **Analista:** Emanuele 🔎

### Scope

**Incluso:**
- Pagina dinamica `/stocks/[symbol]` con dati del titolo
- Header con ticker, nome, prezzo corrente, high/low, variazione
- Grafico storico con periodi selezionabili (intraday, 1W, 1M)
- Endpoint API per dati storici (time series)
- Estensione provider con metodo per time series
- Link dalla watchlist alla pagina dettaglio

**Escluso:**
- Titoli visti di recente (US-011)
- Notizie contestuali (US-018)
- Note personali (US-016)

### Requisiti Impliciti

- Serve un metodo `getTimeSeries(symbol, interval, outputsize)` nel provider
- Twelve Data ha endpoint `/time_series` con intervalli: `1min`, `5min`, `1day`
- Per intraday: intervallo `5min`, outputsize ~78 punti (1 giorno di trading)
- Per 1W: intervallo `1day`, outputsize 5
- Per 1M: intervallo `1day`, outputsize ~22
- Serve una libreria di charting leggera (Recharts — già compatibile con React)
- La pagina deve funzionare anche senza autenticazione (URL diretto)

### Assunzioni

- Recharts come libreria di charting (leggera, React-native, buon supporto responsive)
- I dati time series di Twelve Data sono disponibili nel free tier
- La pagina non richiede autenticazione (accessibile via URL diretto)

## Soluzione Tecnica

> **Architetto:** Leonardo 📐

### Approccio Scelto

Estendere il provider con `getTimeSeries()`, creare endpoint `/api/stocks/[symbol]/history`, pagina Next.js `/stocks/[symbol]` con componente chart e selettore periodo.

### Motivazione

Pattern coerente con il provider esistente. Recharts è la scelta naturale per grafici in React — zero config, responsive, bundle size contenuto. La pagina è un server component con sezioni client per interattività (chart, selettore periodo).

### Modifiche al Data Model

Nessuna.

### Modifiche alle API

**`GET /api/stocks/[symbol]/history?period=intraday|1w|1m`**
- Nessuna auth richiesta
- Parametri: `period` (default: `intraday`)
- Mapping periodo → Twelve Data: intraday → `5min,78`, 1w → `1day,5`, 1m → `1day,22`
- Risposta 200: `{ symbol, period, data: TimeSeriesPoint[] }`
- Risposta 400: periodo non valido
- Risposta 502/429: errore provider

### Modifiche al Frontend

**Pagina `/stocks/[symbol]`:**
- Header: ticker (grande, mono), nome, exchange
- Card prezzi: prezzo corrente (grande), high/low, variazione abs/%, colorata
- Sezione chart con selettore periodo (tabs: Intraday | 1W | 1M)
- Server component che fa fetch iniziale della quote + passa dati al client

**Componente `StockChart`:**
- Client component con Recharts (AreaChart o LineChart)
- Selettore periodo con tabs/buttons
- Fetch `/api/stocks/[symbol]/history?period=...` al cambio periodo
- Responsive, tooltip con prezzo e timestamp
- Colore linea: verde se variazione positiva, rosso se negativa
