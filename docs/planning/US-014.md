## User Story

**Epic:** EP-005 — Multi-Watchlist e Funzionalità Avanzate
**Priorità:** MEDIUM | **Story Points:** 3

**Story**
As Marco (investitore retail),
I want creare più watchlist con nomi diversi (es. "Tech USA", "ETF Europa"),
so that possa organizzare i titoli per tema o strategia.

**Criteri di Accettazione**
- [ ] L'utente può creare una nuova watchlist specificando un nome
- [ ] L'utente può rinominare o eliminare una watchlist esistente (con conferma per eliminazione)
- [ ] La dashboard permette di switchare tra watchlist con un selettore o tab
- [ ] Ogni watchlist mantiene i propri titoli indipendentemente dalle altre

## Soluzione Tecnica

> **Architetto:** Leonardo 📐

### Approccio Scelto

API REST per CRUD watchlist + selettore watchlist nella dashboard (tabs o dropdown) + adattamento del WatchlistPanel per accettare un `watchlistId` dinamico.

### Motivazione

Il modello dati supporta già watchlist multiple per utente. Serve solo esporre le API CRUD, aggiungere un selettore nella dashboard e parametrizzare il WatchlistPanel. L'architettura esistente non richiede modifiche strutturali.

### Componenti Impattati

| Componente | Tipo Modifica | Descrizione |
|---|---|---|
| `src/app/api/watchlists/route.ts` | Nuovo | GET (list) + POST (create) |
| `src/app/api/watchlists/[id]/route.ts` | Nuovo | PATCH (rename) + DELETE |
| `src/components/watchlist-selector.tsx` | Nuovo | Tabs/dropdown per switchare watchlist |
| `src/components/watchlist-panel.tsx` | Modifica | Riceve watchlistId, fetch titoli specifici |
| `src/app/dashboard/page.tsx` | Modifica | Integra watchlist-selector |

### Modifiche al Data Model

Nessuna modifica — il modello Watchlist e WatchlistItem esistono già con tutti i campi necessari.

### Modifiche alle API

**`GET /api/watchlists`** — Lista watchlist dell'utente autenticato
- Risposta 200: `{ watchlists: Watchlist[] }` (con conteggio items)

**`POST /api/watchlists`** — Crea nuova watchlist
- Body: `{ name: string }`
- Validazione: nome non vuoto, lunghezza max 50 caratteri
- Risposta 201: `{ watchlist: Watchlist }`

**`PATCH /api/watchlists/[id]`** — Rinomina watchlist
- Body: `{ name: string }`
- Validazione: nome non vuoto, ownership verificata
- Risposta 200: `{ watchlist: Watchlist }`

**`DELETE /api/watchlists/[id]`** — Elimina watchlist
- Verifica ownership
- Cascade delete su WatchlistItem (gestito dal DB)
- Risposta 200: `{ success: true }`

### Modifiche al Frontend

**`WatchlistSelector`:** Componente con tabs o dropdown che mostra le watchlist dell'utente. Include:
- Lista watchlist con nome e conteggio titoli
- Pulsante "+" per creare nuova watchlist (dialog con input nome)
- Menu contestuale per rinominare o eliminare (con dialog di conferma per eliminazione)
- Stato selezionato sincronizzato con il WatchlistPanel

**`WatchlistPanel`:** Adattato per ricevere un `watchlistId` come prop. Fetch e rendering dei titoli filtrati per la watchlist selezionata.

**`Dashboard`:** Integra il WatchlistSelector sopra il WatchlistPanel, gestendo lo stato della watchlist attiva.

**Tipo:** Implementazione

**Descrizione:**
Implementare le API REST per la gestione delle watchlist multiple:
- **GET /api/watchlists**: restituisce tutte le watchlist dell'utente autenticato, con `_count.items` per mostrare il numero di titoli in ciascuna

- **PATCH /api/watchlists/[id]**: rinomina una watchlist verificando ownership
- **DELETE /api/watchlists/[id]**: elimina una watchlist verificando ownership (cascade delete su items gestito dal DB)

Tutte le API richiedono autenticazione (401 se non autenticato). Verifica ownership per PATCH e DELETE (404 se watchlist di altro utente).
