## Soluzione Tecnica

> **Architetto:** Leonardo

### Approccio Scelto

Aggiungere checkbox di selezione alla watchlist, creare un endpoint `/api/stocks/compare` che recupera time series per piu simboli in parallelo e normalizza i dati in percentuale, implementare un componente `ComparisonChart` con Recharts (LineChart multi-linea) e una pagina `/stocks/compare` che integra selezione e grafico.

### Motivazione

Riutilizzare `getTimeSeries()` dal provider esistente evita duplicazioni. L'endpoint di confronto aggrega e normalizza lato server, riducendo la logica client. Recharts supporta nativamente linee multiple con colori e legenda. La normalizzazione percentuale rende confrontabili titoli con prezzi molto diversi.

### Modifiche alle API

**`GET /api/stocks/compare?symbols=AAPL,GOOGL,MSFT&period=1d`**
- Nessuna auth richiesta
- Parametri:
  - `symbols` (obbligatorio): lista simboli separati da virgola (min 2, max 5)
  - `period` (default: `1d`): `1d`, `1w`, `1m`
- Mapping periodo:
  - `1d` -> intervallo `5min`, outputsize 78
  - `1w` -> intervallo `1day`, outputsize 5
  - `1m` -> intervallo `1day`, outputsize 22
- Logica: chiama `getTimeSeries()` per ogni simbolo in parallelo con `Promise.all()`
- Normalizzazione: per ogni serie, calcola `((close - firstClose) / firstClose) * 100`
- Risposta 200:
  ```json
  {
    "symbols": ["AAPL", "GOOGL"],
    "period": "1d",
    "data": [
      {
        "datetime": "2026-03-19 09:30:00",
        "AAPL": 0.0,
        "GOOGL": 0.0
      },
      {
        "datetime": "2026-03-19 09:35:00",
        "AAPL": 0.45,
        "GOOGL": -0.12
      }
    ]
  }
  ```
- Risposta 400: meno di 2 o piu di 5 simboli, periodo non valido
- Risposta 502/429: errore provider
