## User Story

**Epic:** EP-005 — Multi-Watchlist e Funzionalita Avanzate
**Priorita:** LOW | **Story Points:** 2

**Story**
As Giulia (studentessa di finanza),
I want aggiungere note personali a un titolo nella mia watchlist,
so that possa annotare le mie osservazioni e motivazioni di interesse.

**Criteri di Accettazione**
- [ ] L'utente puo aggiungere una nota testuale (fino a 500 caratteri) a un titolo in watchlist
- [ ] La nota e visibile nella pagina dettaglio del titolo e/o come tooltip nella watchlist
- [ ] L'utente puo modificare o eliminare una nota esistente
- [ ] Le note sono private e visibili solo all'utente che le ha create

---

## Analisi dei Requisiti

> **Analista:** Emanuele

### Scope

**Incluso:**
- Campo `note` opzionale sul modello `WatchlistItem` (max 500 caratteri)
- API PATCH per aggiornare/eliminare la nota di un item in watchlist
- Inclusione della nota nella risposta GET della watchlist
- Tooltip/popover nella lista watchlist per visualizzare/editare la nota
- Sezione nota nella pagina dettaglio del titolo
- Validazione lunghezza massima (500 caratteri) lato API e UI

**Escluso:**
- Note condivise con altri utenti (fuori scope)
- Formattazione rich text / markdown (fuori scope)
- Allegati o immagini nelle note (fuori scope)
- Storico modifiche delle note (fuori scope)

### Requisiti Impliciti

- Il modello `WatchlistItem` esiste gia con relazione a `Watchlist` e `Symbol`
- La privacy e garantita dal fatto che ogni `WatchlistItem` appartiene a una `Watchlist` che a sua volta appartiene a un `User` — la query filtra sempre per `userId`
- La cancellazione della nota equivale a impostare il campo a `null`
- Il conteggio caratteri deve essere visibile in tempo reale nell'UI

### Assunzioni

- Il modello `WatchlistItem` in `prisma/schema.prisma` e disponibile e funzionante
- Le API watchlist saranno implementate o estese per supportare l'update degli item
- La pagina dettaglio titolo sara disponibile per integrare la sezione nota

## Soluzione Tecnica

> **Architetto:** Leonardo

### Approccio Scelto

Aggiungere un campo opzionale `note String?` al modello `WatchlistItem` in Prisma, creare un endpoint `PATCH /api/watchlist/items/[id]` per aggiornare la nota (e restituirla nelle GET), e implementare componenti UI per visualizzare e modificare la nota sia nella watchlist (tooltip/popover) che nella pagina dettaglio titolo.

### Motivazione

L'approccio piu semplice e diretto: un singolo campo `note` sul modello esistente evita la creazione di tabelle aggiuntive. Il PATCH endpoint segue le convenzioni REST per aggiornamenti parziali. Il popover nella watchlist offre accesso rapido senza navigare altrove, mentre la sezione nella pagina dettaglio consente editing piu comodo.

### Modifiche al Data Model

**Modello `WatchlistItem`** — aggiungere:
```prisma
note String? @db.VarChar(500)
```

Migrazione: `npx prisma db push`

### Modifiche alle API

**`PATCH /api/watchlist/items/[id]`**
- Auth richiesta (verifica che l'item appartenga a una watchlist dell'utente)
- Body: `{ "note": "testo della nota" }` oppure `{ "note": null }` per eliminare
- Validazione:
  - `note` deve essere stringa o `null`
  - Lunghezza massima 500 caratteri
  - L'item deve esistere e appartenere all'utente autenticato
- Risposta 200: item aggiornato con nota
  ```json
  {
    "id": "uuid",
    "watchlistId": "uuid",
    "symbolId": "uuid",
    "sortOrder": 0,
    "note": "La mia analisi su questo titolo...",
    "symbol": { "ticker": "AAPL", "name": "Apple Inc." }
  }
  ```
- Risposta 400: nota troppo lunga o formato invalido
- Risposta 401: utente non autenticato
- Risposta 403: item non appartiene all'utente
- Risposta 404: item non trovato

**Includere `note` nelle risposte GET esistenti della watchlist** — quando le API watchlist saranno disponibili, assicurarsi che il campo `note` sia incluso nel select/include di Prisma.

### Modifiche al Frontend

**Componente `NotePopover`:**
- Icona nota accanto a ogni titolo nella watchlist (icona piena se nota presente, vuota se assente)
- Click apre un Popover (shadcn/ui) con:
  - Textarea per la nota (max 500 caratteri)
  - Contatore caratteri in tempo reale (es. "142/500")
  - Bottone "Salva" (PATCH con il testo)
  - Bottone "Elimina" (PATCH con `null`), visibile solo se nota esistente
- Loading state durante il salvataggio
- Toast di conferma su successo

**Componente `NoteSection`:**
- Card nella pagina dettaglio titolo
- Textarea editabile con contatore caratteri
- Bottoni "Salva" e "Elimina"
- Stesso comportamento del popover ma con layout piu ampio
- Messaggio placeholder se nessuna nota presente ("Aggiungi una nota...")
