## User Story

**Epic:** EP-002 — Quotazioni Near-Real-Time
**Priorità:** HIGH | **Story Points:** 5

**Story**
As Marco (investitore retail),
I want che le quote si aggiornino automaticamente senza dover ricaricare la pagina,
so that possa monitorare i movimenti in tempo quasi reale durante la sessione.

**Criteri di Accettazione**
- [ ] Le quote si aggiornano automaticamente a intervalli configurabili (default suggerito: 15-30 secondi)
- [ ] Un indicatore visivo segnala quando un refresh è in corso
- [ ] Il sistema rispetta i limiti del piano gratuito Twelve Data (rate limiting lato server)
- [ ] Il refresh si interrompe automaticamente quando il mercato è chiuso o il browser tab non è attivo
- [ ] L'utente può forzare un refresh manuale con un'azione esplicita

## Analisi dei Requisiti

> **Analista:** Emanuele 🔎

### Scope

**Incluso:**
- Polling automatico delle quote a intervallo configurabile (default 30s)
- Indicatore visivo di refresh in corso (spinner, pulse animation)
- Rate limiting lato server per proteggere il free tier
- Pausa automatica quando tab non visibile (Page Visibility API)
- Pausa quando il mercato USA è chiuso (orari NYSE/NASDAQ)
- Bottone di refresh manuale

**Escluso:**
- WebSocket o SSE (troppo complesso per il free tier, non necessario)
- Configurazione utente dell'intervallo (hardcoded con possibilità di override futuro)

### Requisiti Impliciti

- Rate limiter lato server per il provider Twelve Data (8 req/min nel free tier)
- Logica di market hours USA: NYSE aperto lun-ven 9:30-16:00 ET, escluse festività
- Il polling deve fermarsi e ripartire quando il tab torna attivo
- Il refresh manuale bypassa il timer ma non il rate limiter

### Assunzioni

- US-006 (batch quotes) è completato — il polling riusa lo stesso endpoint
- Il free tier Twelve Data ha 8 API calls/min e 800/day
- Un intervallo di 30s con 20 ticker = ~2 chiamate/min (entro i limiti)

## Soluzione Tecnica

> **Architetto:** Leonardo 📐

### Approccio Scelto

Hook personalizzato `useQuotePolling` che gestisce il ciclo di polling con setInterval, pausa su tab hidden / mercato chiuso, e refresh manuale. Rate limiter semplice lato server nell'endpoint batch quote.

### Motivazione

Un hook React custom è il pattern più pulito per gestire il ciclo di vita del polling. Il rate limiter server-side protegge le credenziali API indipendentemente dal comportamento del client.

### Componenti Impattati

| Componente | Tipo Modifica | Descrizione |
|---|---|---|
| `src/hooks/use-quote-polling.ts` | Nuovo | Hook per polling automatico con pausa/resume |
| `src/lib/market-hours.ts` | Nuovo | Utility per determinare se il mercato USA è aperto |
| `src/lib/rate-limiter.ts` | Nuovo | Rate limiter in-memory per l'API Twelve Data |
| `src/app/api/quotes/batch/route.ts` | Modifica | Aggiungere rate limiting |
| `src/components/watchlist-panel.tsx` | Modifica | Integrare useQuotePolling, indicatore refresh, bottone manuale |

### Modifiche al Data Model

Nessuna.

### Modifiche alle API

- `GET /api/quotes/batch`: aggiungere rate limiter in-memory (token bucket, 8 req/min)
- Risposta 429 quando il rate limit è superato, con header `Retry-After`

### Modifiche al Frontend

**Hook `useQuotePolling`:**
- Input: `symbols: string[]`, `interval: number` (default 30000ms)
- Output: `{ quotes, isRefreshing, lastUpdated, refresh, isMarketOpen }`
- Logica: setInterval che chiama il batch endpoint
- Pausa: `document.visibilitychange` → ferma intervallo su hidden, riprende su visible
- Pausa: `isMarketOpen === false` → non fa polling, mostra stato
- Refresh manuale: funzione `refresh()` che chiama immediatamente

**WatchlistPanel:**
- Sostituire il fetch quote singolo con `useQuotePolling`
- Indicatore di refresh: spinner/pulse quando `isRefreshing === true`
- Bottone refresh manuale
- Stato mercato: badge "Mercato chiuso" quando `isMarketOpen === false`
