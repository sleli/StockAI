## User Story

**Epic:** EP-006 — Notifiche e Integrazioni Esterne
**Priorità:** MEDIUM | **Story Points:** 5

**Story**
As Marco (investitore retail),
I want ricevere una notifica email quando un alert si attiva,
so that possa essere avvisato anche quando non ho l'app aperta.

**Criteri di Accettazione**
- [ ] Quando un alert si attiva, il sistema invia un'email all'indirizzo associato all'account dell'utente
- [ ] L'email contiene: titolo, ticker, soglia impostata, prezzo che ha causato il trigger e timestamp
- [ ] L'utente può attivare/disattivare le notifiche email dalle impostazioni del proprio profilo
- [ ] Il sistema non invia email duplicate per lo stesso trigger (deduplicazione)

## Soluzione Tecnica

> **Architetto:** Leonardo 📐

### Approccio Scelto

Estendere il modello User con campo `emailNotifications`, creare un servizio email con Resend, integrare l'invio nel flusso di trigger alert esistente, e tracciare le notifiche inviate con un modello `EmailNotification` per la deduplicazione.

### Motivazione

Resend offre un SDK TypeScript nativo che si integra perfettamente con Next.js, ha un tier gratuito adeguato (100 email/giorno), e supporta template React per le email. La deduplicazione tramite tabella DB è più robusta di approcci in-memory e sopravvive ai restart.

### Modifiche al Data Model

**Modello User — aggiunta campo:**
```prisma
model User {
  // ... campi esistenti
  emailNotifications Boolean @default(true)
  emailNotificationsSent EmailNotification[]
}
```

**Nuovo modello EmailNotification:**
```prisma
model EmailNotification {
  id        String   @id @default(uuid())
  userId    String
  alertId   String
  sentAt    DateTime @default(now())
  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  alert     Alert    @relation(fields: [alertId], references: [id], onDelete: Cascade)

  @@unique([alertId])
  @@index([userId])
  @@index([alertId])
}
```

Il vincolo `@@unique([alertId])` garantisce che non si possa inviare più di una notifica per lo stesso alert trigger (deduplicazione a livello DB).

### Modifiche alle API

**`GET /api/settings`** — Recupera preferenze utente

**`PATCH /api/settings`** — Aggiorna preferenze
- Body: `{ emailNotifications: boolean }`
- Risposta 200: `{ emailNotifications: boolean }`

### Modifiche al Frontend

**Componente NotificationSettings:** Toggle switch (shadcn/ui Switch) con label e descrizione. Fetch GET /api/settings al mount, PATCH al cambio.
