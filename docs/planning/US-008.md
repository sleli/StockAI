## User Story

**Epic:** EP-002 — Quotazioni Near-Real-Time
**Priorità:** MEDIUM | **Story Points:** 3

**Story**
As Giulia (studentessa di finanza),
I want sapere chiaramente se il dato che vedo è aggiornato, vecchio o non disponibile,
so that possa fidarmi delle informazioni mostrate e capire quando il mercato è chiuso.

**Criteri di Accettazione**
- [ ] Ogni quota mostra uno stato visivo: fresh (dato recente), stale (dato non aggiornato da oltre 5 min), market closed, error
- [ ] Un banner o toast informa l'utente quando il limite API del provider è stato raggiunto
- [ ] Lo stato "market closed" viene determinato in base agli orari di apertura del mercato USA (NYSE/NASDAQ)
- [ ] In caso di errore, viene mostrato l'ultimo dato disponibile con indicazione chiara che non è aggiornato

## Soluzione Tecnica

> **Architetto:** Leonardo 📐

### Approccio Scelto

Componente `QuoteStatusBadge` che riceve lo stato calcolato e mostra un badge colorato. La logica di calcolo dello stato viene centralizzata nel `WatchlistPanel` usando i dati dal hook `useQuotePolling`. Un sistema di banner/toast basato su shadcn/ui `Sonner` (toast) per notifiche rate limit ed errori.

### Motivazione

Separare il badge (presentazione) dalla logica di stato (calcolo) permette di riusare il badge in altri contesti e di testare la logica indipendentemente. Il toast di shadcn/ui è già disponibile nel boilerplate e fornisce un pattern UX consolidato.

### Modifiche al Frontend

**Utility `getQuoteStatus`:**
- Input: `lastUpdated: Date | null`, `isMarketOpen: boolean`, `hasError: boolean`
- Output: `"fresh" | "stale" | "closed" | "error"`
- Logica:
  - Se `hasError` → `"error"`
  - Se `!isMarketOpen` → `"closed"`
  - Se `lastUpdated` è null o più vecchio di 5 minuti → `"stale"`
  - Altrimenti → `"fresh"`

**Componente `QuoteStatusBadge`:**
- Props: `status: "fresh" | "stale" | "closed" | "error"`
- Rendering: Badge shadcn/ui con colori e icone:
  - `fresh`: verde, icona check — "Aggiornato"
  - `stale`: giallo/ambra, icona clock — "Non aggiornato"
  - `closed`: grigio, icona moon — "Mercato chiuso"
  - `error`: rosso, icona alert — "Errore"

**Componente `RateLimitBanner`:**
- Mostra un toast/banner quando il polling riceve errore 429
- Include tempo stimato di ripristino (da header `Retry-After`)
- Si nasconde automaticamente dopo il ripristino

**WatchlistPanel — Fallback:**
- Mantiene in stato l'ultimo set di quote valide (`lastValidQuotes`)
- In caso di errore dal hook, mostra `lastValidQuotes` con badge "error"
- Indicazione testuale: "Ultimo aggiornamento: [timestamp]" evidenziata
